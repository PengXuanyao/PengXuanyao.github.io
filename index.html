大创答辩稿

各位评委老师好，我是基于物联网的智能养殖综合系统设计的负责人彭宣尧，接下来由我代表我们团队做结题答辩；

我将从这四个方面分别介绍我们的项目；

首先是项目背景的介绍；

禽类养殖业是我国养殖行业中占比巨大、市场发展潜力大、规模化养殖需求大的行业，其在我国现代化农业体系中占据着关键的地位。本次项目基于该行业背景并结合当下的物联网技术，开发了这一套智能养殖的控制和监控设备，助力该产业集约化、自动化和无人化的发展。

然后是我们的项目概要：

本项目是基于NB-IOT技术的物联网智能养殖项目，主要可以用于不同规模大小的禽类养殖场，特别时对于在人力资源比较匮乏、较偏远的，但又适宜集约化禽类养殖的地区，通过这一套系统的部署，能够对养殖的环境进行时实的检测和控制，并且通过多节点部署，可以针对特定的环境和规模，拓展出不同位置或者数量的节点，实现定制化的环境检测目的。主要功能有：智能环境检测、环境参数控制、微信小程序远程显示与小程序远程控制和节点化部署等。

本系统一共包括环境检测端、硬件控制端、物联网传输端、阿里云平台端、小程序用户端和反馈控制端这些组成部分。其中，环境检测端主要完成的工作是将氨气、温湿度、光照的环境信息进行采样；硬件控制端的主要工作是基于STM32主控芯片实现整体系统的检测和控制功能；物联网传输端主要是发送节点环境信息和接收来自云平台的信息；云平台端主要将接收的信息进行双向转发；小程序端是用户交互接口，进行信息接收和控制；反馈控制端是各个环境调节执行器及其驱动。

接下来是项目的主要内容：

首先是硬件平台，主控芯片我们选用了STM32F407单片机，其工作最高主频为168MHz，满足性能要求；选用的温湿度传感器是DHT11；选用的氨气传感器为MQ-135；选用光照度为BH1750；三者分别通过单总线、ADC转化和I2C总线与主控芯片进行通讯；物联网模块选用的是基于NB-IOT技术的BC26模块，其与单片机通过串口进行通讯，与云平台通过MQTT协议进行传输；云平台租用的是阿里云；控制模块是使用的直流电机风扇和加湿器模拟实际情况。针对于偏远地区电力可能不稳定的问题，我们也设计了电池和直流电源双供电的方案。能够在断电的情况下，维持数天的环境检测功能。

然后是软件部分，关于传感器和物联网模块的驱动程序，我们主要参考其数据手册和基本通讯协议进行设计；然后是固件程序的设计，首先传感器模块、物联网初始化，然后在主循环中，首先判断上一次循环物联模块网接收的信息，执行响应的操作，再采集和处理传感器信息，检测物联网连接状态，然后进行数据的接收和发送；接下来是服务器部分，其整体的业务结构示意图如下，分为执行端、中继端、业务端。执行端为下游的物理系统，中继段包括设置上报消息的器皿设置，由于接收节点信息，同时设置服务器订阅的topic列表，在设备端监听即可获得服务器下达的指令。

业务端是一个完整的web框架，包括数据层、逻辑层、视图层。

下图是数据层的数据库E-R图，其由6个表构成，多级分层，可方便地实现多个节点的数据管理；每个表为一个数据访问对象，每个对象都有的四个方法，实现基本的访问和存储。

逻辑层有下图所示意，包括可以实现的逻辑功能，查看环境参数、设备列表、发送控制指令等。

视图层是小程序的交互界面，包括数据查看页、设备管理页和阈值控制页。（阈值控制还没有完善）

最后是系统的实际演示：

查看数据接收情况、打开关闭设备；更换节点，查看数据情况、打开关闭设备。

最后是发展与展望：

这是我们做出的一个原型系统，未来还有许多可以改进的方面。比如可以加入更多的传感器，应用于更复杂的环境；加入更加完善的控制策略，实现更优的控制策略等等；

我的答辩到此结束，谢谢大家。

目前禽类养殖业的集约化养殖规模愈发趋于庞大，为在提高生产效率的同时，提高生产质量、保证禽类安全等方面在成本合理的条件下，还需要进一步提升改进。在集约化养殖禽类时会经常出现CO2、NH3等中毒的危险，环境温度、湿度易失衡的情况，以及粪便等肥料资源未得道充分利用的问题；并且禽类养殖需要恒定在一定合适的温度、湿度、氧气浓度、光照强度以及噪声强度等条件下，因此，我们基于现实情况，制定了适宜并且现实可行的养殖策略，并基于STM32单片机、以及云端处理等技术进行实现。

项目实施目的是为远程监控养殖场环境并且进行自动化控制，减少人力物力投入，实时检测和反馈环境参数，利用单片机技术对环境进行较低成本、较高精度的测定，并且结合云技术对信息进行进一步的处理、传输和及时反馈，提高集约化管理与生产的质量，提高生产效率和产品品质。实现在较低生产成本的条件下，对养殖场环境进行自动控制与远程监控，并且可以进一步将养殖场移至较为西部偏远地点进行无人化监控，减少占地使用成本等。

项目意义：

1.提高禽类养殖生产效率并且保证生产的质量和品质。

2.提高测量环境参数的准确程度，提高测量的有效性和快速性。

3.提高整体环境系统的稳定性，保证禽类生活条件的舒适与稳定。

4.减少禽类发病率、加快禽类发育速率。

5.减少人力物力投入、促进资源节约型生产。

本项目所设计的系统，一共包括环境检测端、硬件控制端、物联网传输端、阿里云平台端、小程序用户端以及反馈控制端。其中，环境检测端对环境信息进行时实的检测，得到多种环境信息的输入信号，并将环境输入信号送入到硬件控制端进行处理，识别当前的环境状态，再将处理之后的环境信息通过到物联网传输端，由NB-IOT协议传输至阿里云平台端，阿里云平台实现消息的实时转发，即将信息实时反馈至小程序用户端，用户端同时也能够发送控制指令至阿里云平台然后转发至设备进行时实的控制。

通过设计这一分布式、可拓展的物联网智能禽类养殖综合系统，用户能够通过微信小程序端直接对远程的禽舍环境进行监控和控制，并且能够做到对不同规模的养殖产业进行节点化、分布式的部署，进行节点化管理，有效地降低了人力物力成本，切实提高经济效益。

彭宣尧负责团队固件编写的工作，作为负责人协调大家；赵超同学完成了项目的很大一部分工作，发挥了关键的作用；李江龙同学完成了团队全部小程序开发和绝大部分的线上部署工作，为软件方面的编写工作负责到底；陈有祥同学在团队中负责了硬件相关的工作.

在本项目中，设计和实现了一种基于物联网的智能禽类养殖综合系统。其具有传输距离远、环境适应性强、不同规模可部署等突出的特点和优点，并设计了微信小程序，用户能够通过便捷地登录小程序，时实地查看养殖场环境信息，可以对其进行远程的操作控制。基于分布式的系统设计，整套设备能够部署形成网络化结构，各个节点可以根据实际的养殖场情况进行部署，实现客制化、定制化的检测系统。

在实际应用中，基于NB-IOT的通信机制该系统的响应时延较短，响应速度也能够满足系统的要求（秒级），并且能够设定阈值对系统进行自动调节和监控。良好的交互界面和较低的成本开销也使得其具有一定的市场竞争力。

同时，该系统是一个还可以进一步优化和拓展的原型，还有一些不足和可以改进的地方。例如，可以依据实际情况加入更多的传感器进行检测和扩充，设计出更加完善的调控方案。在未来，也能够基于该系统的设计方案做出更全面的系统原型和应用方案。

该项目在具体实施过程中，遇到的困难主要是经验和准备不足，在推进项目的过程中，推进的进度情况往往比计划的相对较慢。该项目目前制作出来了物理原型，但是相关的论文发表和专利申请等事项还没有落实。需要进一步加强项目的管理能力，切实做到对项目进度的把控和推进。

在项目中遇到并解决的关键性问题包括：

  1. 对养殖场的温度、湿度、氨气浓度、光强等环境传感器信息采集；
  2. 对环境传感信息进行处理，进行分析、控制和报警；
  3. 将获得的数据上传到网络云平台以方便监管、调控。
  4. 在各种环境因素变化的情况下，构建一个能够增强系统稳定性，提高养殖场生产效率中心处理系统。
  5. 实际调研养殖场的需求，提高系统的稳定性。
  6. 基于云平台的远程监控。

主要用于电路模块、主控板、传感器、模型制作、学习书籍等购买。

uCOS简介

μC/OS-III（Micro C OS Three 微型的 C 语言编写的操作系统第三版）是由 Micrium 于 2009 年推出的一款可升级的、可固化的、开源的、基于优先级的抢占式实时内核。它提供了 高档实时内核所需要的所有功能，例如任务管理、时间管理、信号量、事件标志组、互斥信 号量、消息队列、软件定时器、内存管理等。μC/OS-III 对任务数量、任务大小、优先级数 量无限制，只限制于处理器所能提供的内存大小。μC/OS-III 也提供了很多其他实时内核中 所没有的，比如能在运行时测量运行性能，直接发送信号或消息给任务，任务能同时等待多 个信号量和消息队列。μC/OS-III 被设计用于 32 位处理器， 但是它也能在 16 位或 8 位 处理器中很好地工作。

μC/OS-III 是一个软件实时内核，主要负责任务的管理和任务间的交流，任务的管理也 叫做多任务处理，其作用是协调和切换多个任务依次享用 CPU，它能使 CPU 的利用率达到 最大，让我们从宏观上感觉是多个 CPU 在同时运行，但是在微观上，对于单核处理器，同 一时刻只能处理一个任务，即任务间是串行执行的。把系统功能分成多个在独自的循环体内 运行的小任务，可以实现模块化管理，程序员可以简单的维护和升级产品。
